Section 01

Quick start — wake up your local

The fastest path from a cold laptop to seeing the launcher in your browser. Use this when you've been away for a few days and forgotten the routine.

1

Open the project folder in VS Code

Open VS Code, then File → Open Folder and select:

C:\Users\Tshepho Joel\Documents\SME SA\2026\Mockups

Make sure Mockups is the workspace root — not its parent.

2

Install Live Server (one time only)

In VS Code: Ctrl + Shift + X to open Extensions, search for "Live Server" by Ritwick Dey, click Install.

3

Start Live Server

Right-click any .html file in the Explorer panel → Open with Live Server. Or click the "Go Live" button in the bottom-right status bar.

4

Open in your browser

Live Server uses port 5500 by default:

  • http://localhost:5500/index.html — the launcher
  • http://localhost:5500/control.html — the controller

Port 5500 already in use?

No drama — Live Server picks the next available port automatically. Check the bottom-right VS Code status bar for the actual URL (e.g. :5501).

Fallback: Python one-liner

If Live Server misbehaves, open a terminal at the Mockups folder and run:

python -m http.server 5500

Then browse to http://localhost:5500/

"No folder open" error

Live Server is fussy — you must open the Mockups folder as the workspace root, not a parent folder like 2026. File → Close Folder → reopen the right one.

Launcher

index.html

The public-facing dev/stakeholder view. Shows finalised mockups in collapsible sections, greys out non-finalised ones, and lists external dev/staging URLs in a separate Development section.

  • Deployed to GitHub Pages
  • Reads from status.js
Private

Controller

control.html

Your private command centre. Git-ignored and never deployed. Passphrase Tshepho@96. Lets you change statuses, assign groups, publish, and manage dev links.

  • Passphrase-gated, local-only
  • Publishes status.js direct to main
  • Token stored in localStorage only

Section 03

The mockup status lifecycle

Every mockup has one of four statuses. The launcher renders each differently.

Not started

No work has begun. Launcher hides it entirely or shows a placeholder.

Under work

Currently designing. Greyed-out on the launcher and non-clickable.

Under review

Ready for your review. Visible but flagged on the launcher.

Finalised

Approved & live. The only status that is clickable from the launcher.

Only Finalised mockups are clickable. Everything else is gated by gate-links.js — clicks show a "Coming soon" toast.

Section 04

Publishing from the controller

How a click in control.html becomes a live update on the launcher.

1

Open control.html & enter passphrase

Passphrase: Tshepho@96 (case-sensitive).

2

Change statuses & assign groups

Use the dropdowns per mockup row. The Group dropdown moves a page between launcher sections.

3

Click Publish

The top-right button commits status.js directly to main via the GitHub API using your saved fine-grained token.

4

GitHub Pages updates in ~1–2 minutes

The launcher auto-reflects the new state. No build step.

5

If publish fails

Open Settings in the controller → paste a fresh GitHub PAT. Stored only in localStorage — never committed.

Bypass note

The controller commits directly to main. The repository's Branch Ruleset has you on the bypass list — that's why direct commits work without PRs.

Section 05

Dev build links (staging)

When a developer has shipped a real page, you can point reviewers to it — even before the mockup itself is finalised.

In the controller, set a mockup's Dev status (Pending review / Reviewed / Approved) and paste the staging URL (e.g. https://smesastg.wpengine.com/).

On publish, the launcher's Development section lights up the live link. Clicking opens the staging URL in an iframe panel for side-by-side review.

Dev links bypass the mockup gate — useful when the dev has shipped but the design isn't finalised yet.

Section 06

The link gate

How non-finalised pages stay quietly hidden from internal navigation.

  • gate-links.js runs on every finalised page

    It reads the current status of every page from status.js on load.

  • Non-finalised links are neutralised

    Any anchor pointing to a non-finalised page is intercepted — click shows a "Coming soon" toast and no navigation happens.

  • Flipping to finalised activates links instantly

    The moment you publish a page as finalised, all inbound links across the hub activate automatically. No re-deploy needed.

Section 08

Common tasks

The everyday flows you'll do over and over.

  • Mark a mockup as finalised

    Controller → status dropdown → Finalised → Publish.

  • Move a mockup to a different section

    Controller → Group dropdown → pick new group → Publish.

  • Add a custom group

    Controller → + GROUP button → name it → Publish.

  • Add a dev/staging link

    Controller → Dev URL field + Dev status → Publish.

  • See current state of all mockups

    Just open the launcher (index.html).

  • Compare a dev build vs its mockup

    Click the Compare button next to the dev link in the launcher's Development section.

Section 10

Troubleshooting

When something breaks, scan this list before panicking.

Live Server won't start
Close VS Code completely, reopen the Mockups folder as the workspace, try Go Live again. If it still fails, check Extensions → Live Server isn't disabled.
Controller says "invalid passphrase"
The passphrase is Tshepho@96 — case-sensitive. Check for a stray space, wrong shift key, or autocorrect.
Publish failed: "bad credentials"
The GitHub token has expired or was rotated. Controller → Settings → paste a fresh fine-grained PAT with Contents: write scope for SMEZATech/smehubmockups.
"Coming soon" toast on every link
Those target pages are not yet Finalised. Open the controller, flip their status, then publish.
Dev link iframe is blank
The staging host blocks embedding (X-Frame-Options / CSP). Use the "Open dev" external-link button next to the iframe instead.
Launcher sidebar all collapsed
That's the default for visual cleanliness. The section containing the active mockup opens automatically — everything else stays tidy until you click.
