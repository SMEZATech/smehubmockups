Colour roles · surface vs text

Colours have roles, not just values. Two dark shades in the palette look almost identical but do very different jobs. Never use a surface colour for text, and never use a text colour for a surface — even when the hex values are close.

The #0C1F31 vs #121A21 trap

SURFACE

#0C1F31 · Navy

Backgrounds only — dark hero sections, dark cards, dark bands, footers.
Never use for text.

TEXT

#121A21 · Ink

Primary typography — H1/H2, body copy, CTAs on light cards.
Never use for a background.

Text on light backgrounds

  • #121A21 — Ink · H1/H2, body, primary text, CTA labels
  • #6A7581 — Muted · byline, date, 'Last updated', card meta
  • #374151 — Reading grey · long-form article body (optional, softer than ink)

Text on dark (navy) backgrounds

  • #FFFFFF — Primary text · H1, body on hero
  • rgba(255,255,255,.75) — Sub copy under H1
  • rgba(255,255,255,.55) — Muted · breadcrumb, 'Last updated', meta
  • rgba(255,255,255,.30) — Very muted · › separators

The rule

The mockup is the source of truth. If you're unsure which token to use, inspect the mockup element with dev tools and copy the exact value. Don't guess and don't 'round' to a similar colour. When a hex looks familiar but a shade off, it's usually the surface-vs-text mismatch above.

✓ Do
  • Use Plus Jakarta Sans 700–800 for headlines and buttons
  • Keep body copy in Inter 400–500
  • Use tight tracking (-1% to -2%) on large display text
  • Let headings breathe — generous line height and spacing
✕ Don't
  • Don't reintroduce Oswald or other condensed faces
  • Don't set long body paragraphs in Jakarta
  • Don't mix in a third typeface
  • Don't use ultra-light weights — the brand reads bold

03 · Logo & architecture

The mark & the family

The SME hexagon mark plus wordmark. Shown here in the 2026 type and colour — the official vector lockup is held by the design team and should be placed from source for production.

✓ Do
  • Keep clear space around the mark equal to the height of the "S"
  • Use the white-on-navy lockup on dark / gradient backgrounds
  • Minimum digital width: 120px for the full lockup
  • Use the mark alone (hexagon) only where the brand is already established
✕ Don't
  • Don't recolour the mark outside brand red or white
  • Don't stretch, rotate, or add effects
  • Don't place the colour mark on a busy photo without a scrim
  • Don't recreate the wordmark in a different font

The editorial brief. Topics, Resources, Funding, Forum, Advice. Consistent family, distinct mark.

Newsletter rules · minimalist by design

  • No thumbnail, no image header. Newsletter issues are typographic — feature images make them feel like articles. The list lives or dies on the headline.
  • Date strip — left column, uppercase 11px, brand grey #6A7581. Acts as the visual anchor for the row.
  • Series + issue badges — black The Weekly badge (or green Funding Alert with the orange bolt) followed by Issue #N in muted grey.
  • Title — Jakarta 700, 19px, two-line clamp. Allowed more space without an image competing for attention.
  • Excerpt — Inter 14px, slate, two-line clamp.
  • Read issue link — red CTA with arrow-right (no border-top divider — newsletter rows are denser than article cards).
  • Hover — left border becomes brand red, row lifts subtly. The interaction does the visual work the thumbnail used to.
  • Mobile — stacks vertically: date → badges + title + excerpt + CTA.

Card rules · the dev checklist

  • Tag positioning — top-left of the card (or top-left of the image if image-led).
  • CTA copy — Templates / Reports / Ebooks = "Download Now" (download icon). Long-form Guides = "Read Guide" (arrow-right icon). Articles use full-card hover (no in-card CTA button).
  • Author byline — only on Article cards. Top-bordered row · 30px gradient monogram avatar (#DC183C → #FF9900) + name in Jakarta bold.
  • Image-led vs image-less — Articles + Resources (when used on a featured/archive slot) have images; Guide CPT cards do not use images by default.
  • Resource sub-type pill colours — Ebook / Report = red (#FDECEF / #DC183C) · Template = green (#E4F5EE / #29A37A) · Resource-Guide = amber (#FFF1E0 / #AE6B0A).
  • Guide CPT (variant C) — uses a single category pill (FUNDING / COMPLIANCE / OPERATIONS) in red. The old orange "GUIDE" pill is removed.
  • Consistency rule — the same card markup must be reused wherever its content type appears (resource cards must look identical on public-resources.html, public-resource-category.html and the related section of public-resource.html).
